[오늘의 작업 요약] 문제 해결 보고서
1. 발생한 문제:
배포된 웹사이트의 제목 화면(TitleScreen)에 표시되는 문구가 이상하게 깨져서 나타남.
하지만 개발자 컴퓨터에 있는 원본 코드 파일(components/TitleScreen.tsx)의 내용은 정상이었음.
2. 문제 원인 진단:
로컬(내 컴퓨터) 코드와 원격(GitHub) 저장소의 코드가 일치하지 않는 동기화(synchronization) 문제로 판단.
즉, 정상적인 코드가 git을 통해 GitHub에 제대로 전달되지 않았고, 배포 서비스(Vercel)는 예전 버전의 코드를 가져가서 웹사이트를 만들었기 때문에 발생한 현상.
3. 해결 과정 (Git을 이용한 '강제 새로고침' 전략):
1단계: 파일에 의도적인 '작은 변화' 주기
문제가 된 components/TitleScreen.tsx 파일의 맨 아랫줄에 빈 줄을 하나 추가하고 저장.
목표: Git이 이 파일을 '수정된 파일'로 확실하게 인식하도록 만들기 위함.
2단계: Git 명령어 3단계 정확히 실행하기
git add .
수정된 파일을 포함한 모든 변경사항을 Git의 추적 대상으로 지정.
(중간에 LF will be replaced by CRLF 경고가 나왔으나, 이는 운영체제 간의 줄바꿈 차이를 Git이 자동으로 맞춰준다는 알림이므로 무시했음.)
git commit -m "Fix: Force update title screen content"
변경 내용을 '제목 화면 콘텐츠 강제 업데이트'라는 메시지와 함께 내 컴퓨터에 최종 저장(확정).
git push
내 컴퓨터에 저장된 최신 내용을 GitHub 원격 저장소로 밀어넣기(업로드).
4. 최종 결과:
git push 명령어가 성공적으로 실행되자마자, Vercel이 GitHub 저장소의 최신 코드를 감지하고 자동으로 새로운 배포를 시작함.
잠시 후, 웹사이트에 접속하여 제목 화면의 문구가 정상적으로 표시되는 것을 확인함으로써 문제가 완전히 해결되었음을 확인.
핵심 교훈:
"내 컴퓨터 코드는 정상인데 배포된 사이트만 이상할 때"는 git add . -> git commit -> git push 과정을 다시 한번 정확하게 실행하여 최신 코드를 강제로 동기화하면 대부분 해결된다.
